# 环境配置说明

## 本项目不需要conda！

本项目是一个纯Python项目，只需要标准的Python环境和pip即可运行。

## 系统要求

- **Python版本**：3.8 或更高版本
- **操作系统**：Windows / Linux / macOS
- **依赖包**：python-docx

## 安装步骤

### 方法1：直接安装（最简单）

```bash
# 安装依赖
pip install -r requirements.txt
```

### 方法2：使用虚拟环境（推荐）

使用Python自带的venv模块创建虚拟环境：

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate

# Linux/macOS:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 方法3：使用其他虚拟环境工具

如果你喜欢使用其他工具，也可以：

```bash
# 使用 virtualenv
pip install virtualenv
virtualenv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/macOS

# 使用 pipenv
pip install pipenv
pipenv install

# 使用 poetry
poetry install
```

## 依赖包说明

本项目只依赖一个外部包：

```
python-docx>=0.8.11
```

**python-docx**：用于创建和修改Word文档的Python库。

## 运行项目

安装依赖后，直接运行即可：

```bash
python md2doc.py
```

## 验证安装

运行以下命令验证安装是否成功：

```bash
# 检查Python版本
python --version

# 检查依赖包
pip list | findstr python-docx  # Windows
pip list | grep python-docx     # Linux/macOS

# 运行测试
python tests/test_converter.py
```

## 常见问题

### Q1: 为什么不用conda？

A: 本项目非常轻量，只需要一个Python包（python-docx），使用pip完全足够。conda环境比较重量级，对于简单的Python项目来说不是必需的。

### Q2: 我可以继续用conda吗？

A: 可以！如果你习惯使用conda，完全可以继续使用：

```bash
conda create -n md2doc python=3.12
conda activate md2doc
pip install -r requirements.txt
```

但推荐使用更轻量的venv。

### Q3: 如何卸载？

A: 如果使用虚拟环境，直接删除虚拟环境文件夹即可：

```bash
# 先退出虚拟环境
deactivate  # Linux/macOS
# 或关闭命令行窗口

# 删除虚拟环境文件夹
rm -rf venv  # Linux/macOS
rmdir /s venv  # Windows
```

### Q4: 依赖包安装失败？

A: 尝试以下方法：

```bash
# 升级pip
python -m pip install --upgrade pip

# 清除缓存
pip cache purge

# 使用国内镜像源（加速）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```
